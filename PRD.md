# PRD: 我的核心资产 (iOS)

## 1. 背景与定位

“我的核心资产”是一款面向个人使用的 iOS 投资理财应用，核心理念是：

- 只关注少量“核心资产”，排除其他投资干扰
- 在低估时买入并长期持有
- 在**极度高估**时卖出（非频繁交易）

应用目标是把“纪律化的仓位管理 + 估值区间提醒 + 交易记录”做成一个可长期使用的个人工具。

## 2. 目标与非目标

### 2.1 目标 (Goals)

- 能快速看到整体资产情况与每个核心资产的状态（估值、仓位、持仓、价格）
- 能完成一次性初始化：初始资金、资产清单、理想买入/卖出价、已有持仓
- 能记录日常买入/卖出，自动更新持仓与现金
- 价格自动更新，并支持手动刷新
- 支持多市场/多币种资产，统一换算为人民币做汇总

### 2.2 非目标 (Non-goals)

- 不做券商接入/自动下单
- 不做复杂技术分析指标与短线策略
- MVP 阶段不做云同步/多端协作
- MVP 阶段不追求覆盖全部标的，先从“常用 Top100 + 常关注列表”起步

## 3. 用户与核心场景

### 3.1 用户画像

- 单用户（本人）
- 长期价值投资者，持仓集中，关注少量标的
- 主要诉求：看板化管理、纪律执行、简单可靠

### 3.2 核心场景 (MVP 闭环)

1. 资产看板 (Dashboard)
2. 初始化设置（初始资金 + 资产录入 + 估值参数 + 持仓初始化）
3. 日常交易记录（买入/卖出）

## 4. 核心概念与计算规则

### 4.1 货币与汇总口径

- 单个资产主要显示：**原币种价格**（例如港股显示 HK$，美股显示 $）
- 单个资产次要显示：折算后的人民币市值
- 总资产/仓位/现金等汇总：**统一以人民币**统计

### 4.2 仓位定义

- 当前仓位 = 单个资产当前市值（人民币）/ 当前总资产市值（人民币）
- 目标仓位：用户为每个资产手动设置的理想仓位比例（%）
- 仓位上限：可选，用户设置的单资产最大仓位比例（%），超出时看板提示
- 看板展示当前仓位与目标仓位的对比，便于判断是否需要调仓

### 4.3 估值区间 (5 档)

用户为每个资产手动维护两条基准线：

- 理想买入价 A（原币种）
- 理想卖出价 B（原币种）

当前价格为 P（原币种），缓冲系数固定为 **15%**：

1. 极度低估：P ≤ A × 0.85
2. 比较低估：A × 0.85 < P ≤ A
3. 合理：A < P < B
4. 比较高估：B ≤ P < B × 1.15
5. 极度高估：P ≥ B × 1.15

估值状态在看板以颜色区分（语义化）：

- 极度低估：绿色（强烈买入）
- 比较低估：浅绿（可以买入）
- 合理：灰蓝/中性色（观望）
- 比较高估：橙色（可以卖出）
- 极度高估：红色（强烈卖出）

## 5. 功能需求 (Functional Requirements)

### 5.1 首页资产看板 (Dashboard)

#### 5.1.1 总览区

- 当前总资产（人民币，大字号突出显示）
- 盈亏（金额 + 百分比，带涨跌箭头）
- 已投入金额（人民币）
- 剩余现金（人民币）
- 最近更新时间（价格更新时间）

#### 5.1.2 资产列表

每条资产卡片展示（看板保持清爽）：

- 资产名称 + 市场（A 股/港股/美股/BTC/基金）
- 当前价格（原币种，自动更新，大字号突出显示）
- 仓位（%）+ 目标仓位（%）+ 仓位偏离提示
- 估值状态（5 档之一）

以下信息在**资产详情页**展示：
- 资产代码（symbol）
- 持有数量、平均成本（原币种）、当前市值（人民币）、浮动盈亏
- 理想买入价 / 理想卖出价

#### 5.1.3 排序

- 支持资产列表排序（简化版，MVP 不做筛选）
- 排序选项（选择即生效，无需确认按钮）：
  - 仓位从高到低
  - 仓位偏离目标从大到小（推荐，优先显示需调仓的资产）
  - 估值低估优先

#### 5.1.4 快捷操作

从资产卡片进入：

- 买入
- 卖出
- 编辑资产（理想价、目标信息、备注、是否常关注等）
- 查看交易历史/资产详情

#### 5.1.5 空状态

- 无资产时引导进入初始化/添加资产
- 无交易时引导记录第一笔交易

### 5.2 初始化设置

初始化为 **4 步向导**（V1.1 起，后三步全部支持跳过；只有第 1 步必填）：

1. **设置初始资金**（必填）
2. **添加核心资产**（可零选）— 若不选任何资产，按钮文案变 "完成初始化"，点击直接结束向导
3. **设置估值参数与目标仓位**（可整步跳过）— 顶部蓝色提示卡告知未填也能进；已填字段需自洽（卖出 > 买入）
4. **初始化持仓**（可整步跳过）

设计立意：让用户先用起来，再渐进填写。已添加未设理想买卖价 / 目标仓位的资产，会在看板和详情页显示弱引导（蓝色提示卡），点击 push 到资产编辑页随时补完。

#### 5.2.1 初始资金

- 默认 1,250,000 RMB
- 可修改

#### 5.2.2 资产添加与搜索

支持资产类型：

- A 股
- 港股
- 美股
- 比特币
- 基金（对接中国市场基金数据源，MVP 可先占位）

搜索要求：

- 支持首字母缩写/模糊搜索（例如 “TX” -> “腾讯控股”）
- 支持中文名称搜索（例如 “腾讯”）
- 支持代码搜索（例如 “00700”）

数据源策略（MVP）：

- **方案 A：本地预置 Top100 常用资产列表**（启动快、离线可用）
- 预置“常关注资产列表”入口（默认写入若干常用标的，后续可编辑）

选择资产后自动填充：

- 代码
- 交易所/市场
- 币种

#### 5.2.3 价格与估值参数

- 当前价格：不手动输入，由系统自动获取并更新
- 理想买入价 A：手动输入（原币种），**V1.1 起非必填**
- 理想卖出价 B：手动输入（原币种），**V1.1 起非必填**
- 理想买入/卖出价：**支持修改**
- 未设置时（A=0 || B=0）资产卡片显示"估值未设置"灰色标签 + 弱引导提示，估值色条灰色化

#### 5.2.4 目标仓位设置

- 目标仓位比例（%）：用户为每个资产设定的理想持仓占比，**V1.1 起非必填**
- 仓位上限（%，可选）：超出时看板醒目提示
- 支持随时修改
- 未设置时（targetPositionRatio == 0）卡片显示"未设目标"，仓位偏离提示也相应隐藏

#### 5.2.5 持仓初始化

用户可录入当前已持有仓位：

- 平均成本（原币种）
- 持有数量

币种处理：

- 录入/交易时按原币种
- 系统按汇率折算为人民币做汇总

#### 5.2.6 资产删除

- 支持删除已添加的资产（左滑触发删除）
- 删除前需二次确认（确认弹窗）
- 删除资产时，关联的交易记录自动级联删除
- 删除后剩余现金不做回退（已发生交易视为历史事实）

### 5.3 日常交易记录

#### 5.3.1 买入

字段：

- 资产
- 买入价格（原币种）
- 买入数量
- 买入时间（默认当前时间，可修改）

效果：

- 生成一条交易记录
- 更新持有数量、平均成本（加权）
- 更新现金（人民币）：按汇率折算后扣减

#### 5.3.2 卖出

字段：

- 资产
- 卖出价格（原币种）
- 卖出数量
- 卖出时间（默认当前时间，可修改）

效果：

- 生成一条交易记录
- 更新持有数量（减少）
- 更新现金（人民币）：按汇率折算后增加
- 确认按钮使用红色（语义化：卖出=红色）

#### 5.3.3 交易历史

- 按时间倒序展示，按日期分组
- 区分买入/卖出（绿色/红色标签）
- 每条记录布局：
  - 第一行：资产名称 + 买卖标签 + 金额（原币种）
  - 第二行：价格 × 数量 + 交易时间
- 空状态：引导前往看板查看资产详情并交易

### 5.4 价格更新与刷新

#### 5.4.1 自动刷新（MVP 已实现）

- **进入看板页**时立即自动刷新一次
- 之后每 **10 秒**自动刷新一次
- 仅在看板 Tab 激活时运行，离开看板自动停止
- 自动刷新静默执行，不弹提示（失败也不打扰用户）

（V0.2 可细化为开盘时段 5 分钟、非开盘 1 小时的分时策略。）

#### 5.4.2 手动刷新

- 支持在首页**下拉**或**右上角按钮**触发立即刷新
- 手动刷新失败时弹提示告知用户

#### 5.4.3 刷新调度机制

- 统一入口 `triggerRefresh(source:showMessage:)` 管理所有刷新请求
- 防重入：同一时刻只允许一个刷新任务运行，避免并发重复请求
- 刷新任务独立于 SwiftUI 视图生命周期，降低被系统取消的概率

#### 5.4.4 错误处理与降级

- 真实 API 失败时降级为 MockPriceService（±2% 随机波动）
- 请求被系统取消（-999 NSURLErrorCancelled）单独识别，不按 API 失败处理
- 单个资产获取失败不阻塞其他资产更新
- 全部失败时提示”刷新失败，请稍后重试”

### 5.5 价格/估值通知（V1.2 已实现）

价格刷新成功后判断每个资产是否跨越关键阈值，跨越则发本地通知。无需后台刷新——依赖前台进入看板时的自动刷新（10s 周期）触发。

#### 5.5.1 触发场景（4 种）

- **价格跌破理想买入价**：`oldPrice > A && newPrice ≤ A`
- **价格突破理想卖出价**：`oldPrice < B && newPrice ≥ B`
- **进入极度低估**：旧档位 ≠ deepUndervalued && 新档位 == deepUndervalued
- **进入极度高估**：旧档位 ≠ deepOvervalued && 新档位 == deepOvervalued

未设理想价的资产不参与跨越检测。

#### 5.5.2 防抖与权限

- 同一资产同类提醒 24 小时内最多发一次（`Asset.alertStateJSON` 持久化最近一次发送时间）
- 首次开启通知时调用 `UNUserNotificationCenter.requestAuthorization`；用户拒绝后在设置页显示警告行 + 一键跳系统设置
- App 在前台时也显示 banner（实现 `UNUserNotificationCenterDelegate.willPresent` 返回 `[.banner, .list, .sound]`）

#### 5.5.3 设置入口

- "我的" Tab 加 "通知" Section
- 主开关 + footer 一段话说明触发场景；4 种 AlertType 的子开关 UI 不暴露（默认全开）
- 提供"发送测试通知"按钮，方便用户验链路

### 5.6 分红/拆股自动检测 + 送股/配股手动入口（V1.3 已实现）

设计前提：用户日常不会盯盘，所以纯手动录入分红/拆股不可行。改为**半自动**——App 主动拉派息/拆股事件，与本地交易比对，找出未录入候选 → 用户在详情页一键确认才落库。**永远要用户确认，不偷偷写数据。**

#### 5.6.1 自动检测范围

- **现金分红**（cashDividend）— 各市场都能拉
- **拆股**（split）— 主要服务美股/港股；A 股拆股极少

送股、配股因免费 API 不可靠，**仅手动录入**（详情页底部"更多"菜单）。

#### 5.6.2 数据源策略

| Market | 主源 | 兜底 |
|---|---|---|
| US | Yahoo Finance v8（events=div,split） | — |
| HK | Yahoo Finance v8（symbol → `0000.HK`） | — |
| CN | 东方财富 datacenter API | 新浪财经 openapi |
| FUND | 东方财富基金分红接口 | — |
| BTC | — | 不分红 |

复用 RealPriceService 的网络模板（10s timeout、重试 2 次、-999 取消识别、静态 URLSession）。失败时静默——不阻塞其他功能，候选卡不显示。

#### 5.6.3 比对/去重

- 取本地 BUY 交易最早 occurredAt 作为 since（限定检测起点）
- 用 dayKey（YYYY-MM-DD UTC）与本地已记录的 DIVIDEND/SPLIT 交易去重
- 同一资产 24h 内复用上次检测结果（`Asset.lastDividendCheckAt`）

#### 5.6.4 用户交互

- 资产详情页顶部 priceCard 之后插入金色"候选事件卡"：
  - 标题"检测到 N 笔未记录事件" + 列出 ≤3 条预览
  - 单条点击 → 弹对应录入 sheet（值已预填，用户可改）
  - "全部记录" 按钮 → 弹 confirm dialog 显示**摘要**（事件数 / 现金净变化 / 平均成本变化预览 / 持仓变化预览）→ 用户确认才批量写入
- 详情页底部由"买入/卖出"二等分改为"买入/卖出/更多"三等分
- "更多" Menu 项：分红 / 拆股 / 送股 / 配股 → 各自打开录入 sheet（无预填）

#### 5.6.5 数据应用公式

- **现金分红**：`cash += perShare × shares × fxRate`；`averageCost -= perShare`（"摄薄成本"语义，数学上不双计——avgCost × shares 等于剩余净投入资金）
- **拆股**：`holdingQuantity *= ratio`；`averageCost /= ratio`；现金不变
- **送股**：`bonus = oldQ × ratio`；`newQ = oldQ + bonus`；`averageCost = oldCost × oldQ / newQ`；现金不变
- **配股**：`averageCost = (oldCost × oldQ + price × newQ) / (oldQ + newQ)`；`holdingQuantity += newQ`；`cash -= price × newQ × fxRate`

#### 5.6.6 撤回机制（不做）

V1.3 不做单笔交易撤回。如果数据出错，用户可整体删除资产（资产管理 Tab 左滑删除），现金会按所有交易反向冲销恢复。删除后重新加资产、重录交易作为兜底逃生通道。

## 6. 数据与模型 (Data Requirements)

### 6.1 Asset（资产）

字段：

- id
- name
- symbol/code
- market (CN/HK/US/BTC/FUND)
- currency (CNY/HKD/USD/...)
- idealBuyPrice (A，原币种；0 = 未设置)
- idealSellPrice (B，原币种；0 = 未设置)
- currentPrice (P，原币种)
- lastPriceUpdatedAt
- holdingQuantity
- averageCost (原币种)
- targetPositionRatio（目标仓位比例，%；0 = 未设置）
- maxPositionRatio（仓位上限，%，可选）
- isWatched / watchListRank（常关注列表）
- notes（可选）
- alertStateJSON（V1.2，通知 24h 防抖；JSON `[AlertType: lastSentDate]`）
- lastDividendCheckAt（V1.3，分红检测 24h 缓存）

### 6.2 Transaction（交易）

- id
- assetId
- type — V1.3 起：`BUY` / `SELL` / `DIVIDEND` / `SPLIT` / `BONUS_SHARE` / `RIGHTS_ISSUE`
- price（原币种，语义按 type 复用：BUY/SELL = 交易价；DIVIDEND = 每股股息；SPLIT/BONUS_SHARE = 比率；RIGHTS_ISSUE = 配股价）
- quantity（语义按 type 复用：BUY/SELL = 交易数量；DIVIDEND = ex-date 持仓数；SPLIT = 净增量；BONUS_SHARE = 送股数；RIGHTS_ISSUE = 配股数）
- occurredAt（DIVIDEND/SPLIT/BONUS_SHARE 是 ex-date；RIGHTS_ISSUE 是缴款日）
- fxRateUsed（可选，记录折算时的汇率，便于复现）
- cnyAmount（折算人民币金额；DIVIDEND 正数=入账；RIGHTS_ISSUE 负数=流出；SPLIT/BONUS_SHARE = 0）

### 6.3 Portfolio（组合/全局）

- initialCashCNY（默认 1,250,000）
- currentCashCNY
- lastGlobalRefreshAt
- hasCompletedSetup（初始化向导完成标记）

## 7. 外部依赖与接口 (Integrations)

### 7.1 行情与汇率

原则：

- 优先免费、稳定、可接受延迟
- 能覆盖 A/H/US/BTC（基金可后置）

MVP 实现（已接入）：

- A股/港股主源：腾讯行情 web.sqt.gtimg.cn（HTTPS，免费免 Key，GB18030 编码）
- A股兜底：Yahoo Finance（000333.SZ / 600519.SS 格式）
- 港股兜底：Yahoo Finance（0700.HK 格式）
- 美股：Yahoo Finance v8 API（HTTPS，JSON，免费）
- 加密货币：Binance data-api（HTTPS，JSON，免费免 Key）
- 基金：天天基金 fundgz API（JSONP，免费）
- 汇率：静态汇率（CNY=1, HKD=0.92, USD=7.20）
- 降级策略：真实 API 失败时降级为 MockPriceService（±2% 随机波动）
- 网络稳健性：统一 timeout=10s、reloadIgnoringLocalAndRemoteCacheData、User-Agent；最多重试 2 次；请求使用 dataTask+continuation 减少 UI 取消连带中断

### 7.2 资产搜索数据源

- 本地 `Top100` 预置 JSON（MVP）
- 后续可追加在线搜索（新浪财经等）作为增强

## 9. 技术方案 (Tech)

### 9.1 技术栈

- iOS 原生：SwiftUI
- 本地存储：SwiftData
- 网络：URLSession + 轻量网络层

### 9.2 约束

- 单机离线可用（除行情/汇率拉取外）
- 数据可靠：本地持久化，避免丢失

## 10. 非功能需求 (NFR)

- 性能：打开首页 1s 内完成首屏（不含网络刷新）
- 可靠性：行情失败不影响浏览本地数据
- 可维护性：行情/汇率/搜索适配器可替换
- 可观测性（本地）：记录最近一次刷新结果与错误信息（便于排查）
